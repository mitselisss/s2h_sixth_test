This is the updated version of the Switch to Healthy app after the bug tracking of the internal testing

The most important functionalities fixed in this version (also can be seen in Asana) are:

1. Not able for a user to register with the same e-mail
2. When creating the default-user-profile, all fields are set 
3. When a user change only the 'daily plans presented in' there is no need to update the plans for the current and next week
4. When no changes have been made in the currently loaded profile screen the 'Update' button should be inactive
5. When the next week plan does not exist arrow should be inactive
6. Disable Moroccan cuisne selection in the user profile
7. Improve the presentation of ingredients in the dish and meals panel
